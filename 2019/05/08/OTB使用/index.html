OTB 使用总结

tracker_benchmark_v1.0是OTB（object tracking benchmark）一个版本

阅读本文前先简单阅读tracker_benchmark_v1.0中的readme。

以下为具体使用方法：

0

下载源码,并解压，得到tracker_benchmark_v1.0。

下载vlfeat工具，解压，并将代码做如下修改：

%mian_running.m

%addpath(('D:/vlfeat-0.9.14/toolbox'));
addpath(('/home/hongda/Desktop/tracker_benchmark_v1.0/vlfeat-0.9.21/toolbox')); %路径对应解压位置

1

1.1

将用来对比的tracker放入tracker_benchmark_v1.0/trackers文件夹下。
将使用的图像序列放入与tracker_benchmark_v1.0/util/configSeqs.m中指定的位置下。

1.2

按照configtracker.m和configseq.m中的格式在其中添加将要使用的视频序列和tracker。

2

修改文件路径。代码适用于windows平台，如果在linux平台运行需要将如下文件中涉及到文件路径的部分所有的”\”改为”/“。

  • main_running.m
  • perPlot.m
  • drawResultBB.m
  • genPerfMaat.m

3

执行main_running.m,在results文件夹中生成.mat文件（存储每个trackers的跟踪结果）和perfMat（存储最终用于画图的mat文件）不一样，这个是相对于bounding box的跟踪框数据。

4

执行perfPlot.m文件画出算法评估图，过程中会在perfMat/overall中生成用于画图的.mat文件并在figs/overall中生成算法评估图。

Attention

% main_running.m
evalType='TRE'; %'OPE','SRE','TRE' %指定算法评估方式finalPath = ['./results/results_' evalType '_CVPR13/']; %指定算法运行结果存储位置
eval(funcName); %运行指定一个算法
save([finalPath s.name '_' t.name '.mat'], 'results'); %存储算法运行结果

% perfplot.m

metricTypeSet = {'error', 'overlap'}; %指定评价指标
evalTypeSet = {'SRE', 'TRE', 'OPE'}; %指定算法评估方式
rankingType = 'AUC';%AUC, threshod %指定算法排序指标
thresholdSetOverlap = 0:0.05:1; %指定overlap阈值范围
thresholdSetError = 0:50; %指定位置误差的阈值范围

if strcmp(metricType,'error')&strcmp(rankingType,'AUC') %
    continue;
end

if ~exist(dataName) %如果没有performance Mat file，那么生成它。
     genPerfMat(seqs, trackers, evalType, nameTrkAll, perfMatPath);
end

plotDrawSave(numTrk,plotDrawStyle,aveSuccessRatePlot,idxSeqSet,rankNum,rankingType,rankIdx,nameTrkAll,thresholdSet,titleName, xLabelName,yLabelName,figName,metricType); %绘图并存储

% drawResultBB.m

pathRes = './results/results_SRE_CVPR13/';% The folder containing the tracking results %指定使用SRE结果进行绘图，因此运行该文件前需要确认是否存在SRE结果。
pathDraw = './tmp/imgs/';% The folder that will stores the images with overlaid bounding box 指定输出文件位置
LineWidth = 4; %指定不同tracker对应跟踪框的线宽。
plotSetting; % 指定drawResultBB.m中不同tracker对应跟踪框的线型和颜色。

A1

每次运行前先删除上次perfMat/overall中的文件。

A2

因为最终的算法评估图生成是需要bounding box数据的所以需要在执行perfPlot.m前先执行main_running.m
